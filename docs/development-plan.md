# 開発計画

## プロジェクト概要

- **目的**: 競輪（3連複・3連単・ライン予想）のAI予想システム構築
- **フェーズ1**: CLIバックエンド（精度検証・改善）
- **フェーズ2**: Webフロントエンド実装
- **データ期間**: 直近1〜2年分

---

## フェーズ1: バックエンド / CLI（予想精度を詰める）

### Phase 1-1: 環境構築・データ収集基盤
**目標**: スクレイパーを実装し、過去データをDBに蓄積

#### タスク
- [ ] プロジェクト環境構築（uv, Python 3.12+）
- [ ] SQLiteデータベース設計・作成
- [ ] 競輪ステーション スクレイパー（requests版）実装
- [ ] OddsPark スクレイパー（requests版）実装
- [ ] netkeirin スクレイパー（Selenium版）実装
- [ ] データ統合パイプライン実装
- [ ] 過去1〜2年分のデータ収集実行

#### 収集対象データ
- レース情報（開催場・グレード・日程・距離・天候）
- 選手情報（競走得点・ギア比・最近成績・地域・ライン位置）
- レース結果（着順・タイム・払戻金）
- 並び情報（ライン構成）

---

### Phase 1-2: データ前処理・探索的分析
**目標**: データの品質確保と特徴量の選定

#### タスク
- [ ] データクリーニング（欠損値・異常値処理）
- [ ] Jupyterノートブックで探索的データ分析（EDA）
  - 着順と各特徴量の相関分析
  - ライン別の勝率分析
  - 開催場別の傾向分析
  - ギア比と結果の関係
- [ ] 特徴量エンジニアリング実装
- [ ] 学習データセット構築（train/validation/test分割）

#### 評価指標設計
- **3着内的中率**: 予想した組み合わせに正解が含まれる割合
- **回収率**: 賭けた金額に対する払戻金の割合（100%以上が目標）
- **Precision@3**: 上位3着の予測精度

---

#### CLI使用例
```bash

python -m keirin backtest --start 2025-01-01 --end 2025-12-31
```

## リスクと対策

| リスク | 対策 |
|--------|------|
| netkeirin アンチスクレイピング強化 | 競輪ステーション/OddsParkを代替に。Seleniumのブラウザ偽装強化。 |
| データ品質が低い（欠損・不整合） | 複数ソースでクロスバリデーション。欠損値補完ロジック実装。 |
| 並び情報が取得困難 | 出走表PDFからのパース or スクレイピング対象を変更。 |
| 予想精度が回収率100%未満に留まる | 期待値フィルター強化。高オッズレース特化モデルの検討。 |
