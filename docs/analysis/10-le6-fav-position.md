# ≤6車: 1位指数(pred1)着順別レース選定 × 三連単推奨の検証（2026-06-09）

> ユーザーの問い:「pred1(指数1位)が **①2-3着 / ②4着以下(バスト)** になるレースを事前選定でき、
> そのレースで**三連単の推奨(=黒字)** が取れるか」を、7+の本命バスト検証(`docs/analysis/05`)を踏まえ ≤6車 で検証。

## 結論（要約）
- **ユーザー仮説は ≤6車 で構造的に成立**（7+とは決定的に異なる）。pred1 が1着になるのは **46-49%だけ**＝「1位指数が2-3着/4着以下」の帯は大きい。
- **選定は可能だが既知のupsetレバーと同一**: `r1_prob`(モデル自己信頼度)が単調・AUC0.63(≠1着)/0.65(バスト)。pred1弱の選定 = top3_sum Q1_loose と同じ正体。
- **三連単推奨は「取れる」= 7+の壁(88%天井)が ≤6車 では効かない**。ただし**頑健なのは「pred1,pred2 1-2着BOX→thirds(6点)」のみ**。pred1を3着に降格/完全除外する構築は的中3-8%で単発高配当依存＝非頑健（7+のバスト除外失敗と同じ）。
- **最大の留保**: 全ROIは最終オッズ=上限値。三連単は高配当帯(Q1_低 中央8,210円)に集中＝**朝→確定ドリフトの影響が最大**。trioより低的中・高分散・点数倍。**統計的にtrio基準を明確に上回るとは言えない**（過去「三連単は安定優先で見送り」と同じトレードオフが再出現）。

### 読み取り
1. **7+との決定的差**: 7+で三連単は全条件88%天井（市場効率の壁）。**≤6車では三連単が全条件★再現>100%**＝非効率ポケットの存在を再確認（fieldsize=効率の主因仮説と整合）。
2. **頑健な唯一の勝ち筋＝`pred1,pred2 1-2着BOX→thirds`(6点)**: 三連単中で最高的中(24-40%)・**CI下限が常に>100%**・最大除去後も高ROI。「pred1が1着or2着」両順を拾うのでユーザーの「**1位指数が2着**」ケースを直接カバー。
3. **pred1を3着降格/完全除外は非頑健**: 的中3-8%・CI下限<100%・最大除で割れる（単発高配当依存）。**「pred1がバストするレースを選んでpred1除外で買う」は頑健な推奨にならない**＝7+のバスト除外失敗と同型。
4. **選定の効果**: pred1弱(r1_prob低)に絞ると全構築でROI倍増（trio 253→462%）。だが trio基準も同様に上がる＝**選定はupsetレバーで、三連単固有の新エッジではない**。

### trio基準との比較（重要な留保）
`1-2着BOX` の点ROI(336%)は trio(253%)以上だが、**的中32% vs 55%・6点 vs 3点・CIは trio と重複(統計的に未分離)**。三連単は同じupsetエッジを**高分散・高配当帯で取り直す器**であり、明確な優位ではない。過去の決定「三連単→三連複は安定優先で見送り(的中56→22%分散大)」と同じトレードオフが再出現。

## 判定 / 推奨
- **問い①②の答え**: ≤6車では (a)pred1が勝たない/バストするレースの**選定は可能**（r1_prob・ゾーン）、(b)三連単推奨は**取れる（★頑健>100%）**＝7+の壁は ≤6車 に効かない。**ただし頑健なのは 1-2着BOX のみ**、pred1除外バスト軸は不可。
- **唯一の実装候補 = `三連単 pred1,pred2 1-2着BOX→thirds(6点)`**を opt-in 化（特に SS / pred1弱(r1_prob低)ゾーン）。現行SS(pred1→pred2→thirds 3点)を 1-2着BOX(6点)に拡張すると SS的中19→40%・★頑健維持（SS 2269%/最大除1689%）＝**既存SS層の改良候補**。
- **本番化のゲート（他の≤6車レバーと共通・必須）**: 全て最終オッズ上限値で、三連単は高配当帯集中＝**朝→確定ドリフトの影響最大**。`picks_history`(route='wt')のlive実測＋`snapshot_morning_odds_wt.py --report`のドリフト実測後に判断。点数倍・分散増の許容も要確認。backtestだけで本番化しない。
- **やらないこと**: pred1を3着に降格する構築・pred1完全除外バスト軸（非頑健・単発依存）。
