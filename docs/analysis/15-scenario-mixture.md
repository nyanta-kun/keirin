# ≤6車 展開シナリオ・ミクスチャ検証 — 複数展開の条件付き確率合成（2026-06-11・撤退確定）

> **結論: 撤退。** 展開シナリオは事前に予測可能（Phase1通過）だが、シナリオ条件付きミクスチャ
> P(combo)=Σ P(s|race)·P(combo|s) の組み合わせ確率改善は**統計的に有意でなく**（Phase2不通過）、
> per-combo value も #4（`09-kelly-stake.md`）と同型の壁（較正P≈市場示唆・ROI<<100%）。
> スクリプト: `scripts/exp_scenario_ev_wt.py`

## ユーザーの仮説（問い）

現行モデルは1日のレース全体を統計的に見る＝レースの「平均的な展開1本」。ミクロにはレース単位で
複数の展開（逃げ切り/捲り/番手差し…）があり、真の組み合わせ確率分布は多峰性のはず。
複数展開を予想して総合的に期待値を取れる買い目を出せないか。

## 設計（ゲート方式・撤退ラインを先に設定）

- **シナリオ定義**（勝者の決まり手 `wt_entries.factor` × ライン位置・≤6車で綺麗に4分割）:
  nige=逃(34%) / makuri=捲(22%) / sashi_f=差・非ラインリーダー=番手差し(34%) / sashi_l=差・リーダー(10%)
- **期間**: TR-fit 2023-01〜2025-08 (5,410R) / TR-cal 2025-09〜2026-02 (803R) / TEST 2026-03〜06-10 (513R)
- **公平性**: 本番 `lgbm_wt` はTEST期間学習の可能性 → ベースラインは**同一ハイパラ・同一特徴(FEATURE_COLS_WT)・
  同一期間で学習したプール単一モデル**。較正(isotonic)は両アームともTR-cal（学習外）でfit、判定はTEST。
- **コントロール**: 同サイズ**ランダムレース分割×4のミクスチャ**＝「展開情報」と「アンサンブル容量」の切り分け。
- **ゲート**:
  - Phase1: シナリオ4クラス分類器がTESTで base rate logloss を下回るか
  - Phase2: ミクスチャがプール単一を **オッズ非依存指標**（trio logloss / 的中組合せ-logP）で
    **有意に**（同一レース対bootstrap 95%CI）上回り、かつランダム分割も上回るか
  - Phase3: Phase2通過時のみ value ROI（TR-cal/TEST両方>100%・CI）

## 結果

### Phase1: 展開は事前に予測可能（通過）

| | logloss | base | 改善 | acc | base acc |
|---|---|---|---|---|---|
| TR-cal 803R | 1.2197 | 1.3354 | +0.116 | 44.6% | 34.1% |
| TEST 513R | 1.2404 | 1.3225 | +0.082 | 40.2% | 34.1% |

レースレベル特徴（n_senko・ライン構成・先行最強者のz・S/B回数・上がり戦術率等21個）で base rate 超え。
**展開予測そのものは成立する**（ただし精度40%＝弱いシグナル）。

### Phase2: ミクスチャの改善は有意でない（不通過・判定確定）

較正後のTEST確率品質（オッズ非依存・512R / 9,016 trios）:

| アーム | trio-logloss | race -logP | Brier |
|---|---|---|---|
| プール単一 PL（公平ベースライン） | 0.18410 | 2.3212 | 0.048245 |
| **シナリオ・ミクスチャ** | 0.18345 | 2.3119 | 0.048082 |
| ランダム分割ミクスチャ（容量コントロール） | 0.18373 | 2.3135 | 0.048224 |
| 本番 lgbm_wt PL（参考） | 0.17700 | 2.2096 | 0.046869 |
| **市場示唆 (1/odds正規化)** | **0.16613** | **2.0521** | **0.044773** |

点推定の序列は仮説通り（mix > rnd > pool）だが、対比較bootstrap（同一レース対・4000）で:

- ミクスチャ − プール単一: Δ-logP **-0.0092 [‑0.0333, +0.0142] → 有意でない**
- ミクスチャ − ランダム分割: Δ-logP **-0.0016 [‑0.0245, +0.0204] → 有意でない**（=改善の大半は容量で説明可能）
- 本番lgbm_wt − 市場示唆: Δ-logP **+0.1575 [+0.1045, +0.2098] → 市場が有意に優位**

**決定的な事実は最終行**: どのモデルアームよりも市場オッズの方が組み合わせ確率として有意に較正されている。
シナリオ分解の微小な改善を400倍にしても市場とのギャップ(0.16)に届かない。

### Phase3: per-combo value 無し（参考値・#4と同一の壁）

- 市場効率診断（TEST）: P_model = 0.017 + 0.70·P_market, corr 0.81。
  **P_model>P_market の組: モデル主張 6.2% / 実的中 3.4% / 市場示唆 3.7%** ＝「割安」は錯覚、市場が正しい。
  ミクスチャでも数値はほぼ同一（0.0622/0.0336/0.0371）。
- value ROIスイープ: 全 ev_min で TR-cal 62-78% / TEST 49-65%。★再現ゼロ。
  ミクスチャはプール単一を上回りもしない。

## 解釈

1. **仮説の前段は真**: 展開は多峰的で、事前特徴から部分的に予測できる（Phase1）。
2. **しかし後段が成り立たない**: 展開情報は (a) 既にモデル特徴（line_*/n_senko/style_enc/s・b_count）に
   周辺化されて入っており、(b) 市場はさらに上手く織り込んでいる。条件付き分解で取り出せる
   「市場が知らない多峰性」は残っていない。
3. #4（per-combo value無し）・#7（ライン明示ルール劣後）・doc13①（is_lone_nige不採用）と完全に整合。
   **≤6車の収益はレース選別（top3_sum・fav_mismatch）から来るのであって、組み合わせ確率の精緻化からは来ない**
   という構図が、これで「シナリオ条件付き」経路でも確認された。

## 再開条件

公開オッズ・出走表を超える情報源（並び予想の実測、直前の選手コメント、money-flow等）で
P(s|race) の精度が市場の織り込みを超える見込みが立った時のみ。現行データ内での
シナリオ分割・混合の変種（クラス数変更・決まり手以外のラベル等）は多重比較ノイズであり**追わない**。
