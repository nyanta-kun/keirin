> **主要結論**
> 1. **「pred5 を削る」は逆効果** — pred5 が ROI 111.4%★ で最も収益性が高い（市場の過小評価）
> 2. **O10（個別コンボ≥10倍のみ）が唯一 Phase2 通過** — VAL 112.2%★ / HOLD 100.0%★
> 3. **15〜30倍帯が ROI 205.4%★** — 個別コンボ足切りでここに集中すると高ROI

## 解釈と結論

### なぜ O10 が機能するか

```
現行 S0 の問題:
  pred3 の trio odds が 7〜9 倍台 → 5〜10倍帯(ROI 19〜69%)
  この帯は「市場が正しく高評価した目」= 期待値マイナス
  全3点買いすると、この低ROI目がポートフォリオ全体を引き下げる

O10 の効果:
  5〜10倍の非効率プライシング目をスキップ
  10倍超の「やや意外目」に集中 → 15〜30倍帯(ROI 205%)を取り込む
  pred5 のような「確率が低いが市場が過度に回避している目」を保持
```

### なぜ P2 / G系 が悪化するか

```
pred5 = ROI 111%★ の「隠れた好ターゲット」
P2 / G系 = pred5 を機械的に削除 → 最も収益性の高い目を除去
VAL で 36.7% という壊滅的な結果 → 採用禁止
```

### O15 が HOLD で崩壊する理由

- 15〜30倍帯は hit率 7.7%（約 1/13 レース）= 極めてまれ
- HOLD 11R × 7.7% = 0.84 ヒット期待値 → 実際0ヒット = 確率的な外れ
- VAL 200%★ は 34R × 7.7% = 2-3 ヒットの高オッズ当選
- **O15 は高オッズ宝くじ型 = 信頼区間が極めて広く採用不可**

## 実装提案（O10 live 検証）

### CLI に `--min-combo-odds` オプション追加

```bash
# 現行（レース単位足切り）
wave-picks-wt --gami-skip-odds 3.0 --b-rank-odds 5.0 ...

# 追加（個別コンボ単位足切り）
wave-picks-wt --gami-skip-odds 3.0 --b-rank-odds 5.0 --min-combo-odds 10.0 ...
```

**動作**:
- 各 thirds (pred3/pred4/pred5) について、個別の trio odds を確認
- odds < 10.0倍 の目は購入リストから除外
- 残った目のみで買い目生成（0目になった場合はレーススキップ）

**期待される変化**:
- 推奨レース数: 76R → 約 58R（24%削減）
- 平均買い目数: 3.0点 → 1.4点（53%削減）
- 1日あたりコスト: ~300円/R × 2R/日 → ~140円/R × 1.5R/日 ≈ 50%削減
- 期待ROI: 73% → 112%（目標・上限値）

**live 追跡 picks_history タグ**: `combo_odds_trim`（既存 SS/S/A ランクと別集計）
