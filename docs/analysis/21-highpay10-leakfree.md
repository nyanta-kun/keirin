# 万車券狙い三連単~10点×構成別構造 — リーク無し再検証で不成立（2026-06-12）

> ユーザーの問い: 「三連単で万車券以上を狙う。的中率10-20%目標・1R~10点・出走馬の構成により
> 買い目をわけて購入し、ROI100%超を目指す」
> スクリプト = `scripts/exp_highpay10_leakfree_wt.py`（doc18標準セマンティクス準拠・再利用可）
> 期間: TRAIN 2023-07〜2025-06（選択のみ）/ VAL 2025-07〜2026-02 / HOLDOUT 2026-03〜06-12
> 対象: 出走表基準 n∈{5,6}・全エントリーでランキング・欠車点は返還相当で除外・リーク無しモデル
> （7+は docs/analysis/05,14 でクローズ済みのため対象外）

## 結論

**不成立。全36セル（構造4×HPオーバーレイ2×選別3 ＋ 構成別合成ポリシー2）が判定不通過。**
リーク無し・本番忠実では doc18 と同じ ~50-95% の帯に収束し、万車券狙いも公開オッズ内エッジ無しの
壁の内側だった。さらに本検証で**「的中10-20%」と「万車券中心」は5-6車の市場価格構造上、両立しない**
ことがデータで確認された（下記④）。

1. **doc14 S3（旧TE826%）のリーク無し死亡確認**: 対照F0(=S3)はALLで3期間 75-79%に収束。
   UPSET選別でも TRAIN91/VAL62/HOLD120% と期間間で無一貫＝doc14の高ROIは3バイアスの産物で確定。
2. **的中率10-20%帯は達成可能だがROIが届かない**: F1(市場本命1着外し)が的中14-16%でちょうど目標帯。
   しかしROIは50-78%・的中時中央配当2,000-4,000円帯で、必要水準（的中15%なら平均6,667円）に届かない。
3. **万車券点のみ購入(+HP)は的中率が1-2%に崩壊**: オッズ≥100倍の点だけ買うと10点中の該当が2-5点に
   減る一方、的中はほぼ消える。TRAIN/VALで>100%が出たセル(MM×F0+HP等)は全てHOLD 0%＝0-1的中のノイズ。
4. **構造的不可能性（本検証の主知見）**: 5-6車（全組合せ60-120通り）では
   - 的中10-20%を出す買い方 → 的中の大半が低配当（万車券率5-18%・中央2-4千円）
   - 万車券だけを狙う買い方 → 的中1-2%（10点では確率の絶対量が足りない）
   の二択しかなく、**「的中10-20%×平均獲得≥6,667円」の組合せは市場価格上存在しない**。
   市場はテール（高配当域）も控除率込みで概ね正しく価格付けしている＝7+で確定した市場効率の
   テール版が≤6車にも存在する。
5. **構成別ポリシー（n×gap12でセル分け→セル毎にTRAIN最良構造）も不通過**:
   ALL合成 VAL73/HOLD68%・UPSET合成 VAL86/HOLD95%（的中18-22%）。UPSET合成が最良だが
   CI[49,152]で100%未達かつ払戻=最終オッズ上限値（万車券帯は朝→確定ドリフト最大）のため実力はさらに下。

## 事前登録した設計

- 構造（各~10点）: F0=doc14のS3（対照）/ F1=市場本命を1着から外す10点 / F2=モデル最尤top4順列10点
  （本命型対照）/ F3=2着波乱9点（p1→人気薄2着）
- +HPオーバーレイ: 各構造のうち三連単オッズ≥100倍の点のみ購入（=「万車券以上を狙う」の直接実装）
- 選別: ALL / UPSET(top3_sum≤TRAIN Q1) / MM(fav_mismatch)
- 構成別ポリシー: セル=n(5/6)×gap12三分位(TRAIN固定)。TRAIN ROI最大（制約: 的中≥10%・n≥30）の
  構造をセル毎に選択し、合成をVAL/HOLDで評価
- 判定: リーク無しモデルで VAL/HOLDOUT 両方 ROI>100% かつ 的中10-20%

## 判定 / 今後

- **万車券狙い~10点（構成別含む）は不採用で確定**。変種（セル割り・構造・閾値の組替え）は
  多重比較ノイズになるため追わない（doc15/16と同じ規律）。
- 唯一100%近傍だったUPSET×構成別合成（VAL86/HOLD95%・上限値）も、朝オッズ選別・ドリフトを
  考慮すると実力は下。live実測なしに動かす根拠はない。
- 再開条件は他レバーと同じ: **公開オッズに含まれない情報源**（並び実測・風・money-flow）か、
  picks_history のlive実測での異常確認のみ。
