# 車数制限なし: 高配当レース検知 × 三連単~10点のROI検証（2026-06-11）

> ユーザーの問い: 「高配当の発生するレースの検知、三連単の買い目(~10点)を作りROI確保する方法。車数の制限なしで検討」
> スクリプト = `scripts/exp_highpay_trifecta_wt.py`（再利用すること）
> model=lgbm_wt_eval(OOS)・TRAIN 2023-07〜2026-02 72,534R / TEST 2026-03〜06-08 7,450R（5車以上）
> 払戻=最終オッズ=**上限値**。判定規律: ★頑健 = TRAIN&TEST>100% かつ TEST最大払戻除去後>100%。

## 結論（要約）
1. **万車券(三連単≥10,000円)の事前検知は≤6車で十分可能**: top3_sum AUC **0.732**・Q1_低(波乱帯)で万車券率**51.9%**(基準26.3%)・中央配当10,760円。fav_mismatch(モデル1位≠市場本命)で39.6% vs 一致21.8%。7+は検知力自体が弱い(AUC0.638・Q1でも39%)。
2. **7+は「高配当検知×三連単」でも壁内＝クローズ再確認**: クローズ(docs/analysis/05)後に発見された最後の未検証レバー**fav_mismatchを7+に適用しても TRAIN 78-86%**・全セル(6選別×4構造)でTRAIN<100%・★再現ゼロ。TEのRANK∩MM S2 101%はTR82%＝ノイズ。**7+の再開条件（公開オッズを超える情報源）は不変**。
3. **≤6車は全選別×全構造で★頑健**（24セル全部★）。**~10点の最良構造=S3「3頭BOX＋ヒモ拡張」**: 3頭BOX{p1,p2,p3}(6点)＋p1⇄p2→{4位,5位}(4点)。S1(現行BOX6点)比で**的中率+10〜12pp**(26→38%等)・ROI同等・最大除はむしろ改善。
4. **高配当の取り方は「選別」**: RANK∩UPSET(現行ランク∩top3_sum≤p25)で S3 = **TR532%/TE826%・最大除708%・的中38.2%**・約1.4R/日。RANK∩MM も同水準(TR549/TE852%・0.95R/日)。

- ≤6車のtop3_sum Q1帯は「2レースに1回万車券」＝高配当レースの事前検知は成立。

## 判定 / 推奨
- **答え**: 高配当レース検知→三連単~10点でのROI確保は**≤6車でのみ成立**。
  - 検知 = top3_sum≤p25(波乱ゾーン) ＋ fav_mismatch（万車券率52%/40%帯）
  - 買い目 = **S3: 3頭BOX{p1,p2,p3}(6点)＋p1⇄p2→{4位,5位}(4点) = 10点**
  - 選別 = 現行ランク(gap12≥0.06)∩波乱ゾーン → 約1.4R/日・TE的中38%・ROI最大除でも700%級（上限値）
- **7+は対象外で確定**（本検証で最後の未検証レバーfav_mismatchも消化済・以後車数無制限の再検討は不要）。
- **留保（本番化ゲート・他レバーと共通）**: 全ROIは最終オッズ上限値。本構造は**万車券帯を狙うため朝→確定ドリフトの影響が全戦略中最大**。SS 6点BOX(`--ss-trifecta-box`)のlive前向き検証・朝オッズドリフト実測と同じゲートを通過するまで本番化しない。S3はS1とCI重複＝「6点→10点拡張の価値は的中率」と割り切ること。
