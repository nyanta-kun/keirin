# Web予想ロジック監査: コメント・バンク諸元（2026-06-12・両方Phase1不通過）

> Web上の一般的予想ロジック（doc調査: ライン/番手/競走得点/スジ/バンク/コメント等22種）を保有データと
> 突合した結果、未使用は4つ（コメント・バンク詳細・ギア差分・落車明け・風）。うち2つを検証し両方撤退。
> 判定は全てリーク無しプロトコル（TRAIN 2023-07〜2025-06学習・VAL/HOLD評価・オッズ非依存のAUC/logloss）。

## ① 選手コメント特徴化 — 不通過

- `wt_entries.comment` 68.5万行・充足率~100%（「自力。」16.3万/「○○君。」番手指名/「単騎で。」等）
- 6特徴（自力/自在/単騎/番手指名/補充/レース内自力数）→ **AUC差 ±0.0008以内（全期間×全車数）**
- 理由: wtの並び予想（line_group/line_pos/is_line_leader）がコメントを織り込んで生成されており完全冗長。

## ② バンク諸元（みなし直線長・カント） — 不通過（ただしデータ修繕の副産物あり）

- 全43場+廃止場の周長/みなし直線/カント/屋内を2ソース照合で収集し `venue_info` に追加
  （`straight_len`/`cant_deg` 列・netkeirin Perfecta Navi × keirin-brother照合）。
- **副産物: venue_info.bank_length の大量誤記を発見・修正**（宇都宮333→500・大宮400→500・立川500→400・
  平塚500→400・静岡500→400等、48行更新）。本番特徴 `bank_length_enc` は次回再学習から正値。
  ※誤記修正でもAUC不変＝モデルは周長をほぼ使っていなかった。
- 前提自体はデータで成立: 勝者決まり手は直線<50mで逃28.5%/差41.0% → ≥58mで逃22.7%/差48.8%（単調・9.7万R）。
- しかし素追加もEXPLICITな脚質×バンク交互作用（x_nige_st等4特徴）も **AUC差±0.0004＝無情報**。
  理由: レース定数特徴は相対ランキングに交互作用経由でしか効かず、その効果は既存の
  bank_length_enc×style_enc から木が既に学習済み。決まり手シフト(6-8pp)は実在するが
  「レース内で誰が上位か」をほぼ動かさない。

## 結論と残候補

- 公開情報の再エンコード（コメント・バンク詳細）は情報利得ゼロ。市場どころか**自モデルさえ既に知っている**。
- Web監査の残候補: ギア変更差分・落車明けフラグ（同様に期待薄）、**風・天候（外部収集・唯一の
  「市場が直前まで織り込みにくい」候補・kazekeirin.info等）**。
- 7+の「明確な条件分け」については、ここまでの全証拠（doc05/17/18/19）が
  公開オッズ内では条件分けの効果なしを示しており、再開はやはり外部情報源（風/money-flow/並び実測）のみ。
