## 1. フォーメーション定義と点数

対象条件:
- ≤6車（出走表基準）
- gap12 ≥ 0.06（本番 A 層下限）
- 最安trio オッズ ≥ 5.0 倍（ガミ帯回避）
- 払戻: `wt_odds` 最終オッズ × 100

| フォーメーション | 定義 | 平均点数(ALL) | 平均点数(Q4) |
|----------------|------|-------------|------------|
| current | pivot1-pivot2-{3rd 3点}（trio三連複） | 2.9点 | 3.0点 |
| F1 | pivot1 × クロスライン軸 × {3rd 3点} | 3.0点 | 3.0点 |
| F2 | pivot1 固定 × rank2-5 から C(4,2)=6点 BOX | 5.1点 | 6.0点 |
| F3 | second_line-pivot1-{third 2点}（逆張り） | 2.0点 | 2.0点 |

### 各フォーメーションの詳細

**current（ベースライン）**
```
pivot1 = モデルrank1
pivot2 = モデルrank2
3rd   = rank3/4/5 の三択（3点）
購入: trio(frozenset) × 3
```

**F1（クロスライン軸）**
```
pivot1     = モデルrank1
cross_pivot = pivot1 と異なる line_group から pred_prob 最大の選手
3rd        = rank2-5 のうち pivot1/cross_pivot 以外（最大3人）
購入: trio(frozenset) × 最大3点
```
`line_group` は `wt_entries.line_group`（整数）を直接利用。NULL/欠損の場合はそのレースをスキップ。

**F2（pivot1 固定BOX）**
```
pivot1     = モデルrank1
candidates = rank2/3/4/5（4人）
購入: trio({pivot1, a, b}) for a,b in C(4,2) = 6点
```

**F3（逆張り）**
```
second_line = pivot1 と異なる line_group から pred_prob 最大の選手
3rd        = pivot1/second_line 以外（最大2人）
購入: trio(frozenset({second_line, pivot1, t})) × 2点
「非最強ライン先頭が1着になる崩れ」をターゲット
```

## 4. 考察

### 4-1. upset Q4 絞りが機能しない理由

最大の問題は **サンプル数の極端な少なさ** である。

- UPSET Q4 の定義: top3_sum < 1.71（全エントリーモデル確率上位3頭の合計が極めて低い）
- さらに gap12 ≥ 0.06 かつ MIN_ODDS ≥ 5.0 でフィルタすると、HOLD 期間に 1〜2R しか残らない
- 「Q4 かつ gap12≥0.06」という条件は矛盾を内包している:
  - gap12≥0.06 は「モデルrank1が rank2 を 0.06 以上リード」を要求
  - top3_sum が低いのに gap12 が大きい = rank1 はやや突出しているが rank2/3 は均等に低い
  - このような構造は実際には稀なレース形態

HOLD 期間の Q4 率がわずか 1.6%（= 387R 中 6R）であること、さらに MIN_ODDS フィルタで 1〜2R に絞られることは、この戦略が **本番で発火機会がほぼない** ことを意味する。

### 4-2. フォーメーション間の差異

ALL 条件での VAL × HOLD の点推定 ROI を比較すると:

- **F2（BOX6）がやや高い** (VAL 98% / HOLD 133%) が、CI 下限は 59%/70% で不通過
- **current** は点推定が最も低いが VAL・HOLD ともに ROI が近く安定している（65R/32R）
- **F1（クロスライン）** は HOLD で 74% と最も低く、「クロスライン台頭」シナリオが HOLD 期間には実現していないことを示す
- **F3（逆張り）** は点数が少なく投資効率が低い割に ROI も改善しない

F2 が ALL 条件で TRAIN 120%/HOLD 133% と若干高い傾向はあるが、TRAIN で強い F2 は **単に点数が多く（6点/R）分母が大きい一方で万車券に依存しやすい** 構造であり、CI 上限値（206%）が広いことからも単発依存が疑われる。

### 4-3. doc02 数値との比較

- doc02 では current Q4 ROI = 598%（doc18 バイアスあり）
- 本実験（doc18 修正済み eval model）: current Q4 TRAIN = 92%、VAL = 79%
- バイアス修正で約 86% の数字が消滅した（doc18 の予言通り）
- 「current > 案A > 案B > box5」という doc02 の順序も本実験では再現しない

### 4-4. line_group の利用可能性

`wt_entries.line_group` は整数値（1〜7）として存在し、NULL なし。F1/F3 で「異ライン選手」の選択に直接使用した。line_group の連番は予想発表時のライン番号に対応しており、別ラインの選手を確実に識別できる（近似ではなく厳密な定義）。

## 5. 結論

1. **全フォーメーション不通過**: doc18 セマンティクスで評価すると、現行 current を含む全フォーメーションが VAL・HOLD の CI 下限 100% を達成しない
2. **upset Q4 絞りは統計的に不能**: HOLD 期間のサンプルが 1〜2R に収縮し評価できない。Q4（top3_sum < 1.71）かつ gap12 ≥ 0.06 かつ MIN_ODDS ≥ 5.0 は本番環境での発火機会がほぼない
3. **F2 のわずかな優位は不確か**: HOLD 133%（CI下限 70%）は点推定で current を上回るが、6点/R の高投資・CI 幅の広さから単発依存が疑われ、採用根拠なし
4. **フォーメーション設計より情報源が問題**: ライン崩れを前提とした設計（F1/F3）が current を上回らないのは、「どのラインが崩れるか」の事前予測精度が公開オッズに折り込まれている可能性を示唆する。money-flow 等の優位な情報源なしではフォーメーション工夫に限界がある
5. **推奨**: フォーメーション変更による改善を追わない。波乱対応は現行の top3_sum / fav_mismatch フィルタを維持し、live 実測（picks_history）で判断する
