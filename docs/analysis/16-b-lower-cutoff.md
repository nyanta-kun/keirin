# ランクB閾値緩和 × 買い目オッズ足切り検証（2026-06-12・採用不可で確定）

> **結論: 緩和不可。** 見送り帯（最安目<3倍）・B帯（3〜5倍）・その合成帯は、買い目単位の
> オッズ足切り（≥2/≥3/≥5/≥10/[10,80]）をどう組み合わせても **TEST（2026-03〜06-11 OOS）で
> ROI 100% を超えるセルが存在しない**。TRAINでは有望に見えるセルが複数あるが全てTESTで崩壊
> ＝多重比較の罠そのもので、TRAIN/TEST両立規律が機能した。
> **副発見: 既存レバーD の「B帯=+EV(~116%)」は最新OOSでは再現しない（86%）** — ロードマップ#1
> （Bランク本採用）に対する否定方向の証拠。
> スクリプト: `scripts/exp_b_lower_cutoff_wt.py`

## 問い（ユーザー）

推奨0件日が続く。Bランクの閾値を下げ（見送り帯を取り込み）、買い目単位のオッズ足切りで
鉄板点だけ除けば、量を増やしつつ ROI 100% を超えられるか。

## 設計

- 定義は本番に忠実: `_assign_tier` で SS/S/A 判定。SS=3連単 p1→p2→x 3点 / S・A=3連複 3点
- レース帯 = 3点の最安オッズ（**wt_odds最終=上限値**。朝オッズ判定とのドリフト差に注意）:
  <2 / 2-3 / 3-5(現行B) / ≥5(現行推奨) ＋ 合成帯 <3 / 2-5 / <5 / 全レース
- 足切り = 点単位オッズ: なし / ≥2 / ≥3 / ≥5 / ≥10 / [10,80]（=docs/analysis/06 のレバーB帯）
- TRAIN 2023-07〜2026-02 (4,188R) / TEST 2026-03〜2026-06-11 (456R・103営業日)
- 規律: bootstrap 95%CI・最大払戻1R除去ROI（除）併記。★=TRAIN/TEST両方>100%かつTEST≥30R

## 結果（SS/S/A 全層・抜粋）

| レース帯 | 足切り | TRAIN | TEST | 判定 |
|---|---|---|---|---|
| <2（超鉄板） | 全条件 | 75-80% | 33-74% | ✗ |
| 2-3（鉄板） | なし→≥10 | 92→126% | 79→**65%**（足切りで悪化） | ✗ |
| 3-5（現行B） | なし | 103% | **86%** [59,121] | ✗ 非再現 |
| 3-5（現行B） | ≥5 / ≥10 | 123 / 144% | 90 / 74% | ✗ TRAINだけ有望 |
| ≥5（現行推奨） | なし | **558%** | **492%** 除458% | ★再現（現行の確認） |
| <3（見送り全体） | ≥10 | 103% | 50% | ✗ |
| 2-5（B下限2へ緩和） | ≥5 | 116% | 77% | ✗ TRAINだけ有望 |
| <5（全取込） | [10,80] | 122% | 59% | ✗ |
| 全レース（帯なし） | なし | 248% | 225% | ★だが下記の通り錯覚 |

- 「全レース×★」は**現行≥5帯の超過利益が緩和分の損失を埋めているだけ**。緩和で追加される
  レース群の限界ROIは各帯の行が示す通り 50〜90%＝追加投資分は持ち出し。
- 足切りの一般傾向: **TRAINのROIを上げ、TESTのROIを下げる**（高オッズ点への集中=分散増・
  小標本の好結果がTRAINに残るが汎化しない）。「除」も軒並み<100%で単発依存。
- S/Aのみに絞っても構図は同一（B帯 87%・2-5×≥5 78%）。

## 副発見: レバーDの「B帯=+EV(~116%)」の現在地

- TRAIN期間では 103%（足切り≥5で123%）と方向は再現するが、**最新OOS（103営業日）では
  86% [59,121]**。CIは100%を跨ぐため「B帯=-EV確定」ではないが、**本採用を正当化する証拠は無い**。
- ロードマップ#1（Bランク本採用判断）は live実測待ちだったが、本結果は否定方向。
  Bは現状の「各自判断・推奨合計に含めず」を維持すべき。

## 量問題（推奨0件日）への含意

- 現行≥5帯の購入可能日は**最終オッズ基準で77%**（朝オッズ基準の実運用ではドリフト分さらに少ない）
  ＝0件日は戦略の仕様であり、データ上「量を足してROI>100%を保てる緩和」は存在しない。
- 量を増やす検証済み候補として残るのは引き続き:
  ① 中間オッズ帯[10,80]点集中（docs/analysis/06・★再現・朝ドリフト実測ゲート待ち）
  ② ワイド1点 value型（docs/analysis/12・live検証中）
  ③ fav_mismatch（docs/analysis/13・タグ蓄積中）
  ──いずれも「緩和」ではなく「別の選別軸」。鉄板帯の取り込みは選別の放棄であり逆方向。
