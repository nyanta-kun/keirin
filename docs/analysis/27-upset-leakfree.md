## 1. 波乱モデル性能（AUC）

### 波乱の定義

- 実際に的中した三連複（trio）の払戻 ≥ 2000円 → is_upset=1
- ≤6車で波乱率: **16.1%**（trio払戻中央値: 550円）

**注意: wt_oddsは全組み合わせのオッズを格納しているため、MAX払戻を使うと常に≥2000になる（≤6車で波乱率99.8%の誤り）。実際の的中組み合わせの払戻を使用することで正しい16.1%を得た。**

### AUC結果

| 期間 | AUC |
|------|-----|
| TRAIN | 0.9887 |
| VAL | 0.5674 |
| HOLD | 0.5893 |

VAL/HOLDでAUC≈0.57 = ほぼランダム（0.5）レベル。波乱モデルはTRAIN期間に極度に過学習しており、汎化能力がない。

### 特徴量設計の問題

UPSET_FEAT_COLS（17特徴）はエントリーモデルの予測分布（pred_entropy, pred_top3_sum等）を含む。
≤6車ではレース構造が7+車と大きく異なり、TRAINの分布パターンがOOSに汎化しない。

## 4. 過学習チェック（doc02との比較）

doc02（`docs/analysis/02-upset-prediction.md`）の波乱Q4 ROI 598%/627%との対比:

| 要素 | doc02 | 本実験 |
|------|-------|--------|
| モデル | 全期間学習（リーク） | TRAIN期間のみ（リーク無し） |
| ランキング | 完走者ベース（欠車生存バイアス） | 全エントリー（doc18準拠） |
| ≤6車判定 | 完走者基準（7+混入33%） | 出走表基準（doc18準拠） |
| 波乱ラベル | MAX払戻（≤6車で99.8%が波乱） | 実際の的中払戻（16.1%が波乱） |
| VAL ROI | 627%（bias込み） | Q3 141%（CI=0〜314%・偶発的） |

**doc02の数字は3バイアス＋波乱ラベルの定義誤りが重なった結果であり、全て無効。**

波乱ラベル誤りの詳細:
- ks版 `odds.trifecta_box` は**的中した組み合わせのみ**を格納（1レース1行・中央払戻850円）
- wt版 `wt_odds` は**全組み合わせのオッズ**を格納（1レース平均38組み合わせ）
- MAX払戻で判定すると≤6車では100%が「波乱」扱いになる → 分類器として機能しない

## 5. 結論

### なぜ波乱モデルが汎化しないか

1. **AUC劣化の根本原因**: ≤6車のレースは出走数が少なくレース構造が均質。  
   TRAINで学んだ「7+車の波乱パターン」が≤6車には適用できない。

2. **標本数の問題**: ≤6車全体で約5,600レース（波乱16.1%で約900レース）。  
   VALで50〜90レース/Q、HOLDで10〜22レース/Q — 統計的判断に不十分。

3. **市場効率**: ≤6車でも三連複は市場で適切に価格設定されており、  
   公開情報（競走得点・過去成績・pred_prob分布）から超過収益を得るのは困難。

### 採否

**採用不可**。波乱モデルによる≤6車レースの選別はエッジなし。

doc02の「波乱Q4 → 高ROI」は3バイアス（欠車生存・完走者基準・モデルリーク）と
波乱ラベル誤り（MAX払戻vs実払戻）の合成結果であり、実証的な価値はなかった。

### 今後の注意点

波乱ゲートを再検討する場合:
- 波乱ラベルは**実際の的中trio払戻**（実測：中央値550円・波乱率16%）を使用すること
- wt_oddsのMAX払戻（常に≥2000）は使用不可
- ≤6車に特化した波乱定義（例: 払戻≥2000=16%・≥1500=33%等）の再検討が必要
- live実測（picks_history）による前向き検証が必須
