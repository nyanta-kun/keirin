モデルの仕組み
LightGBM による3着内確率の二値分類
モデル概要
アルゴリズムLightGBM (勾配ブースティング)
タスクBinary Classification
目的変数3着以内に入るか（1/0）
特徴量24 特徴量
CV AUC0.7575 ± 0.0043
学習データ561,053 エントリ / 80,208 レース
予測確率の使い方
  • 各選手に 3着内確率 (pred_prob) を付与
  • 1位確率 − 2位確率 = gap12（AIの確信度の差）
  • 1位確率 ÷ (3/n車) = ratio（期待値比）
  • gap12 と ratio の組み合わせで SS S A に分類
LightGBM を選ぶ理由
  • 表形式データで最高水準の精度
  • 欠損値を自動処理（特徴量が揃わないレースに対応）
  • 特徴量重要度が解釈しやすい
  • 学習・推論が高速

SS / S / A 3段階戦略
6車立て以下 + gap12 / ratio による自動分類
SS ランク
gap12 ≥ 0.15 & ratio < 1.3
3連単
1→2→{3位候補} 3点300円
AIが1位を強く確信（gap12≥0.15）かつ
市場オッズが読みにくいレース（ratio<1.3）
→ 着順固定で高配当を狙う
S ランク
gap12 ≥ 0.15 & ratio ≥ 1.3
3連複
pivot1-pivot2-{候補} 3点300円
AIが1位を強く確信（gap12≥0.15）かつ
期待値が高いレース（ratio≥1.3）
→ 着順不問の3連複で安定回収
A ランク
gap12 ∈ [0.06, 0.15)
3連複
pivot1-pivot2-{候補} 3点300円
AIの確信度がやや高い
（gap12 が中程度）
→ 件数が多く安定した収益源
なぜ 6車立て以下か？
  • C(6,3) = 20通り vs C(7,3) = 35通り
  • 7車以上は組合せ爆発でROI 75.9%（全月赤字）
  • 6車以下のみでROI 518.6%を実現
買い目の構造（jiku2_3）
  • pivot1（1位予想）・pivot2（2位予想）を軸に固定
  • 3位候補を {3位,4位,5位予想} の3車で流す
  • 1点100円 × 3点 = 300円/レース
  • 1日平均 6件 ≒ 1,800円/日の投資規模
