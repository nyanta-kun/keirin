## 現行戦略: 6車立て以下 SS/S/A 3段階ランク

### 指標定義

```
gap12 = AI予測1位確率 − 2位確率   （AIの1-2着確信度差）
ratio = AI予測1位確率 ÷ (3/n_riders)  （期待確率に対する1位確率の倍率）
```

| ランク | 条件 | 買い目 | 買い目内容 |
|--------|------|--------|-----------|
| **SS** | gap12≥0.15 & ratio<1.3 | 3連単 3点 300円 | 1着固定→2着固定→{AI3,4,5位} |
| **S** | gap12≥0.15 & ratio [1.3, 1.6) | 3連複 3点 300円 | 2軸×3頭流し（AI1-2位軸, 3-4-5位流し）|
| **A** | gap12 [0.06, 0.15) | 3連複 3点 300円 | 同上 |
| スキップ | gap12<0.06 または S条件でratio≥1.6 | — | 対象外 |

### 各ランクの狙い

- **SS（市場の盲点レース）**: AI は1-2着を確信しているのに ratio<1.3 ＝ 市場は混戦と判断。
  オッズが高いまま → 的中時の払戻が大きい。1日 0〜1件の超選別。
- **S（AI確信度高）**: gap12大 かつ ratio が中程度。3連複で安定収益。ratio<1.6 で過人気レースを除外。
- **A（中程度の確信）**: gap12が SS 未満の中間域。3連複でバランス型。

## 買い目構造の根拠

### なぜ 2軸×3頭流し（3点）なのか

6車立て（n_riders≤6）では C(6,3) = 20通りの3連複組み合わせ。
平均払戻は約1,027円 → 3点300円の損益分岐は 168円（= 300 ÷ 20 × 100 × 1.12 相当）。
7車立て以上は C(7,3) = 35通りで平均払戻 ~535円、損益分岐 ~712円 → **構造的に採算不可**。

| 方式 | 点数 | 費用 | 的中率 | ROI比較 |
|------|-----|-----|--------|--------|
| BOX3 (1点) | 1点 | 100円 | 低 | 期待損益は低い |
| **2軸×3頭 (3点)** | 3点 | 300円 | 中 | **損益最大化** |
| BOX4 (4点) | 4点 | 400円 | 高 | ROI が下がる（余計な組み合わせを含む）|

BOX4 の {1,3,4} と {2,3,4} は「上位2頭どちらかが3着外」のシナリオ。  
gap12 が大きいとき（= AI が軸を確信）、これらはコスト過多になる。

### SS が 3連単な理由

SS 条件 (ratio<1.3) = 市場が混戦と判断 = 市場オッズが高い = **3連単でも配当が期待できる**。  
また gap12 が大きい = AI は1-2着の順番を見えている確率が高い。

## winticket ルートでのオッズフィルター

`wave-picks-wt` コマンドでは AI 予想後にオッズを参照して買い目を評価できる。

```bash
python -m src.cli.main wave-picks-wt --min-trio-odds 3.0
```

- 3連複の最低オッズが 3.0 倍未満の組み合わせをスキップ
- オッズが低い = 市場が同じ組み合わせを選択済み = 過人気 = 配当価値が低い
- **判断の流れ**: AIが予想 → 市場オッズを確認 → 低オッズなら見直し or 見送り

> オッズをモデルの**入力特徴量に含めない**のは、発売開始前の事前予想との整合性を保つため。

### ガミ回避（オッズ3段階）— 2026-06-08 採用

```bash
python -m src.cli.main wave-picks-wt --gami-skip-odds 3.0 --b-rank-odds 5.0   # 日次cron既定
```

3点(SS=3連単/S・A=3連複)の**最安目の朝オッズ**で3段階に振り分ける:

| 最安目オッズ | 扱い | 意味 |
|---|---|---|
| **< 3倍** | **見送り**（出力しない） | 明確なガミ＝鉄板で当てても損 |
| **3倍 ≤ … < 5倍** | **Bランク**（別枠・購入は各自判断） | 鉄板寄り。妙味薄だが当てれば微益も |
| **≥ 5倍** | **通常推奨**（SS/S/A） | 配当に余裕＝推奨 |

- 根拠（`scripts/analyze_gami_threshold_wt.py`・最終オッズ上限値・train→OOS）: <3倍点を含むレースは集団としてほぼ収支ゼロの鉄板＝turnoverと分散を増やすだけ。レースごと降りると ROI が大きく改善（TEST 全件286%→<5倍除外636%）、**総利益はほぼ維持**しつつ購入Rは半減。
- **「安い目だけカット」より「レース単位の振り分け」が ROI・総損益とも上**（安い点は鉄板レースのサイン）。
- 5倍以上を推奨、3〜5倍未満はBランクとして購入者判断にゆだね、3倍未満は落車クッションも無く見送り。
- Bランクは `推奨合計投資額` に含めない（detail.json では `rank="B"`・`base_rank` に元のSS/S/Aを保持）。
- ⚠️ 朝オッズ基準。朝→直前ドリフトは `snapshot_morning_odds_wt.py --report` で前向き計測中。オッズ不要で同義の代替として `--upset-gate`（`top3_sum`）も利用可。

### 波乱ステーク傾斜（方針A）— 2026-06-08 実装（opt-in）

```bash
python -m src.cli.main wave-picks-wt --gami-skip-odds 3.0 --b-rank-odds 5.0 --stake-tilt
```

波乱スコア `top3_sum`（上位3頭pred_prob合計）で賭け金を傾斜配分（フラットでなく波乱帯に厚く）:

| 波乱帯 | 倍率 | 賭け金(3点) | 意味 |
|---|---|---|---|
| Q1_loose（最も波乱余地大） | ×2 | 600円 | 高ROIゾーンに厚く |
| Q2 | ×1 | 300円 | 標準 |
| Q3 / Q4_chalk（本命堅） | ×0 | 見送り | 低ROI＝張らない |

- 検証（`scripts/exp_stake_tilt_wt.py`・eval model OOS・最終オッズ上限値）: **TEST ROI フラット351% → 傾斜745%**（最大払戻除去640%・train969%/test745%で順序一致）。傾斜を強めるほどROI↑だが件数↓・分散↑。
- カット定数は `strategy_wt.STAKE_TILT_DEFAULT` / `stake_units()`（週次再計測の `upset_cuts_wt.json` に追従）。
- ⚠️ **既定はoff（フラット）**。ROIは上限値・小標本でCI広・**分散増（高配当ゾーン集中＝的中率低）**のため、bankroll余力とlive実測(picks_history)を見て有効化を判断。`top3_sum` はオッズ不要・朝確定で安定。

### ワイド1点（指数1-2位）— 2026-06-09 実装（opt-in・独立プロダクト）

```bash
python -m src.cli.main wave-picks-wt --wide --wide-min-odds 2.5
```

≤6車全件に「指数1-2位ワイド(W12)」を1点100円で出力する独立プロダクト（SS/S/Aの三連複/三連単とは別枠・合計に含めない）。

- **W12が最良の1点**: 的中率はW12(指数1-2位)が最良（≤6車全体59%・W13=47%/W23=37%）。pred_prob 上位2車＝同時top3確率最大ペア。**新モデル不要**。
- **的中率↔オッズは1:1逆連動**（市場効率の壁・`docs/analysis/12`）: 高的中(70-83%)帯は払戻100円＝ガミ常態(ROI70%台)。**「ワイド1点はガミ無し」は誤り**。的中率だけ追うと負ける。
- **`--wide-min-odds` で足切り（value型）**: W12朝オッズ≥2.5〜3.0倍のみ購入 → 的中50-53%・ROI220-271%（最終オッズ上限値・OOS）。高的中・低分散だが黒字化には足切り必須。
- ⚠️ 既定off・本番cron未反映。最終オッズ上限値・N小・朝ドリフト未反映＝**live前向き検証(picks_history rank='WIDE')後に cron 反映を判断**。`notify_results_wt` がSS/S/Aと別集計で自動採点。
